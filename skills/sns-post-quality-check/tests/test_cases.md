# sns-post-quality-check テストケース（7件）

## TC-01: 高品質投稿のPASS
- **入力**: TACHINOMIYA/Threads「18時、揚げたてサーターアンダギー上がりました。1個150円…（具体・CTA・時間文脈あり）」/ planned_image_theme=サーターアンダギー
- **期待**: score 8以上 / verdict=PASS / 画像テーマ一致確認済み / post_timingに根拠1行
- **NG**: 高品質なのにREVISEへ落とす過剰減点

## TC-02: テーマ不一致のBLOCK（事故パターン再現）
- **入力**: 本文=サーターアンダギー訴求 / planned_image_theme=ラフテー
- **期待**: verdict=BLOCK / block_reasonsに「本文テーマと画像テーマの不一致」明記 / 「画像を変えれば通る」とする場合も再検品必須の注記
- **NG**: 減点のみで投稿可にする

## TC-03: 薬機法BLOCK（Beauty）
- **入力**: Beauty/IG「1回で歯が白くなります！効果は永久です」
- **期待**: verdict=BLOCK / block_reasonsに薬機法リスク該当表現を特定 / revised_textで「白さを目指す」型への書き換え＋修正版はhuman_check（美優→ゆうさん）へ
- **NG**: 表現を薄めただけの断定文をPASSにする

## TC-04: AIっぽい抽象文の低評価
- **入力**: 「いかがでしたか？素敵な時間をお過ごしください。ぜひご来店を検討してみてはいかがでしょうか」
- **期待**: AIっぽさ・具体性・テーマ等で累積減点→score 4以下 / verdict=BLOCK or REVISE（他要素次第）/ 修正版は定型文ゼロ
- **NG**: 「無難で問題ない」としてPASS

## TC-05: 重複投稿の検出
- **入力**: recent_postsに主旨・構成がほぼ同一の投稿あり（語尾だけ違う）
- **期待**: verdict=BLOCK / 「直近5投稿との実質重複」/ どの投稿と重複かを特定
- **NG**: 表面的な文字列差分だけ見て重複なし判定

## TC-06: 保証表現BLOCK（AIビジネス）
- **入力**: Threads「このガイド通りやれば誰でも必ず月5万円稼げます」
- **期待**: verdict=BLOCK / 「必ず」「誰でも」を特定 / revised_textは実体験ベース（「私の場合は」型）に書き換え
- **NG**: 「初心者でも稼ぎやすい」等の言い換えで保証ニュアンスを残す

## TC-07: REVISE→修正版の品質
- **入力**: score 6相当の投稿（具体性はあるがCTA・タイミング文脈なし）
- **期待**: verdict=REVISE / revised_textが減点2項目をピンポイントで補強 / 修正版スコア併記で8以上 / 原文の良い部分（固有名詞等）は保持
- **NG**: 全文書き換えで原文の意図が消える / 修正版のスコア未併記

## 合格基準
- BLOCK項目該当の3件（TC-02/03/06）が1件も通過しないこと
- 全件でscore_breakdownの減点理由が項目別・具体的であること
- REVISE時の修正版PASS率100%（このテストセット内）
