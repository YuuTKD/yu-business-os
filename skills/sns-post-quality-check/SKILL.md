# sns-post-quality-check（投稿品質判定Skill）

## 目的
投稿文を14項目で採点し、**人が見なくても事故らない投稿だけ**を自動投稿ラインに通す。
低品質・事業ズレ・画像不一致・薬機法リスクを投稿前に構造的に止める。

## 対象事業・媒体
| 事業 | 媒体 | 特記 |
|---|---|---|
| TACHINOMIYA | Threads / IG / Google投稿 | 観光客・せんべろ訴求 |
| TREE's Catering | Threads / IG / LINE配信 | 法人・夜職の2トーン |
| Tree Beauty | IG / LINE配信 | **薬機法チェック必須** |
| 琉球火鍋 | IG / Google投稿 | 接待・個室訴求 |
| コンサル | Threads / メルマガ | 実績数字の裏付け必須 |
| AIネットビジネス | Threads | **保証表現・誇大表現チェック必須** |

## 使う場面
- 自動投稿キューに入る投稿文の事前検品（全件）
- 投稿候補ストックの一括採点・選別
- スタッフ作成投稿のレビュー（美優のIG運用など）
- theme_match FAIL後の本文修正案の再検品

## 入力スキーマ
```yaml
post_check:
  business: string
  channel: string          # Threads/IG/Google/LINE/メルマガ
  post_text: string        # 投稿本文（必須）
  planned_image_theme: string|null   # 使う予定の画像テーマ
  recent_posts: [string]|null        # 直近5投稿の本文（重複チェック用）
  goal: string|null        # 集客/認知/来店/購入/口コミ
```

## 判定ロジック（10点満点・減点方式）
基礎点10点から、以下で減点。**BLOCK項目に1つでも該当したらスコアに関わらず投稿不可。**

**減点項目（各-1、最大-1まで／複数該当は累積）**
| 項目 | 減点条件 |
|---|---|
| 具体性 | 固有名詞・数字・シーンが1つもない |
| ターゲット明確度 | 誰向けか本文から特定できない |
| 集客導線 | 行動喚起（来店/保存/返信/リンク）がゼロ |
| ブランド一致 | 事業のトーン基準から逸脱 |
| AIっぽさ | 「いかがでしたか」「〜してみてはいかがでしょうか」「まとめ」等の定型 / 抽象形容詞3連以上 |
| テーマの強さ | 1投稿に複数テーマ混在 |
| 今出す意味 | 季節・時間帯・イベントとの接点ゼロ |
| 反応されやすさ | 保存・返信したくなる要素（問いかけ/実用情報）ゼロ |
| 売上導線 | goalが購入/来店なのに商品・予約への道筋がない |

**BLOCK項目（即・投稿不可）**
- NGワード（差別・攻撃・競合誹謗・政治宗教）
- 薬機法リスク（Beauty：「白くなる」「治る」「効果保証」）
- 誇大・保証表現（AIビジネス：「必ず稼げる」「誰でも」「放置で収入」）
- 商品テーマと画像テーマの不一致（planned_image_themeがallowed_post_themes外）
- 直近5投稿との実質重複（本文の主旨・構成が同一）
- 求人・内部告知・テスト文言（"test"・社内用語）の混入

**投稿可否の閾値**
| スコア | 判定 | 扱い |
|---|---|---|
| 8〜10 | PASS | 完全自動投稿OK |
| 5〜7 | REVISE | 修正版を生成→再検品（自動投稿には流さない） |
| 0〜4 | BLOCK | 破棄 or 人間再作成 |
| BLOCK項目該当 | BLOCK | スコア無関係に不可・理由明記 |

## 実行手順
1. BLOCK項目を先に全チェック（該当したら即BLOCK出力・以降の採点は参考値）
2. 減点方式で採点、score_breakdownを項目別に出す
3. REVISEの場合：減点理由を1つずつ潰した修正版を1本生成し、修正版のスコアも併記
4. おすすめ画像カテゴリ・image_theme・投稿タイミングを付与（image-library-managerの在庫と照合可能な形式）
5. 判定・スコア・理由・修正版を出力

## 出力スキーマ
```yaml
result:
  score: int               # 0-10
  verdict: PASS | REVISE | BLOCK
  score_breakdown: {項目: 減点値と理由}
  block_reasons: [string]|null
  revised_text: string|null      # REVISE時必須。修正版スコア併記
  recommended_image: {category, image_theme}
  post_timing: string            # 曜日×時間帯と根拠1行
  target: string                 # 想定ターゲット1行
  cta_improvement: string|null
  human_check: [string]|null     # 薬機法グレー等、人間判断が要る場合のみ
```

## 禁止事項
- 低品質投稿（7点以下）をPASSにすること
- 求人・内部告知・テスト投稿を通すこと
- AIっぽい抽象文を「無難だから」と高評価すること
- 事業とズレた投稿（Beautyアカウントに火鍋宣伝等）を通すこと
- 画像テーマとズレた本文を「画像を変えればいい」で通すこと（画像変更後の再検品必須）
- 誇大表現・薬機法リスク表現の見逃し

## 人間確認が必要なポイント
- 薬機法のグレーゾーン表現（「トーンアップ」等の境界表現）→美優経由でゆうさん
- キャンペーン価格・限定オファーを含む投稿（金額の正誤）
- ネガティブ話題への言及（クレーム対応・閉店告知等）
- 初めての媒体×事業の組み合わせ（トーン基準の初期設定）

## 成功条件
- PASS投稿のみで自動投稿が回り、投稿起因のクレーム・削除がゼロ
- REVISE→修正版のPASS率80%以上（修正ロジックが機能している証拠）
- スタッフが判定理由を読んで投稿文の書き方を学べる（教育効果）

## 失敗条件
- 「良い投稿です」等の理由なき高評価
- BLOCK理由が特定できない出力
- 修正版が原文の言い換えだけで減点理由を潰していない

## 入力例
```yaml
post_check:
  business: TACHINOMIYA
  channel: Threads
  post_text: "沖縄グルメはいかがでしょうか？当店では美味しい料理を提供しています。ぜひお越しください！"
  planned_image_theme: "サーターアンダギー"
  goal: 来店
```

## 出力例
```yaml
result:
  score: 3
  verdict: BLOCK
  score_breakdown:
    具体性: "-1 固有名詞・数字ゼロ（『美味しい料理』のみ）"
    ターゲット明確度: "-1 誰向けか不明"
    AIっぽさ: "-2 『いかがでしょうか』『ぜひお越しください』の定型2連"
    テーマの強さ: "-1 テーマ不在（サーターアンダギーに一切触れていない）"
    今出す意味: "-1 時間帯・季節との接点なし"
    反応されやすさ: "-1 保存・返信要素なし"
  block_reasons:
    - "商品テーマと画像テーマの不一致：本文にサーターアンダギー要素ゼロ、画像テーマ=サーターアンダギー"
  revised_text: |
    18時、揚げたてサーターアンダギーが上がる時間です。
    外カリ中ふわ、1個150円。泡盛ハイと合わせるのが国際通り帰りの定番になってます。
    「サーターアンダギーって夜食べるの？」→むしろ夜が正解です🍺
    （修正版スコア: 9 / PASS）
  recommended_image: {category: food, image_theme: サーターアンダギー}
  post_timing: "平日17:30-18:30（仕事帰り・観光客の夕食検討時間帯）"
  target: "国際通り周辺の観光客・仕事帰りの一人飲み層"
  cta_improvement: "位置情報タグ＋『今日やってます』の営業中シグナルを追加"
```
