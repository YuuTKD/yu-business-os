# sop-writer テストケース（7件）

## TC-01: 口語決定→full_sop変換（撮影依頼）
- **入力**: decision=「アンダギーの写真足りないからゆうとに撮ってDrive入れといて」/ context=「過去に暗い・横向きが使えず」
- **期待**: 必須7要素すべて充足 / stepsが1行1動作 / ng_examplesに過去失敗2つ（暗い・横向き）反映 / deadline未指定→デフォルト提案 / short_instruction併記
- **NG**: 「きれいに撮る」等の曖昧表現の残存

## TC-02: 曖昧語の数値変換
- **入力**: decision=「Catering事例写真をなるべく早くちゃんと整理して」
- **期待**: 「なるべく早く」→具体的期限提案 / 「ちゃんと整理」→分類基準（事業別/テーマ別/フォルダ名）に変換 / 埋められない基準はopen_questionsへ（最大3問）
- **NG**: 曖昧語をそのままSOPに転記

## TC-03: ゆうさん確認過多の検出
- **入力**: decision内に「金額も文言も送信先も全部俺が確認する」を含む営業DM外注依頼
- **期待**: owner_check_pointsを2つに絞る設計提案（例：金額のみ確認＋文言はテンプレ承認制へ）/ 「確認3箇所以上=委任設計見直し」の提案が先に出る
- **NG**: 確認5箇所のSOPをそのまま出力

## TC-04: 外注依頼文の金額プレースホルダ
- **入力**: output_type=outsource_request / 営業リスト50件作成
- **期待**: 検収条件・NG事項・報告様式・修正回数が明記 / 報酬欄=【ゆうさん承認後に記入】/ open_questionsに予算上限確認
- **NG**: 相場からの確定金額記載

## TC-05: 担当者不明の差し戻し
- **入力**: assignee=「誰かスタッフ」/ decision=口コミ依頼の徹底
- **期待**: SOPは仮生成しつつ、assigneeを候補つき質問（ゆうと/店舗スタッフ+選定基準1行）としてopen_questionsに / yes_no_questionは担当確定後に有効と明記
- **NG**: assignee=「スタッフ」のまま完成品として出力

## TC-06: 薬機法配慮の受け手別調整（Beauty）
- **入力**: decision=「美優にIGで施術写真投稿を任せたい」
- **期待**: ng_examplesに薬機法NG表現（「白くなる」等）とお客様許諾の2系統 / 報告テンプレに許諾取得状況の欄 / owner_check_pointsは初回文言確認のみ
- **NG**: 許諾・薬機法への言及なし

## TC-07: short_instructionの品質（LINE1通制約）
- **入力**: output_type=short_instruction / 投稿後スクショ報告の標準化
- **期待**: LINE1通（200字目安）に「やること・形式・期限・報告先」が全部入る / 報告テンプレ（コピペ文）併記
- **NG**: 長文SOPの単純短縮で必須情報（期限・形式）が欠落

## 合格基準
- 全件で必須7要素の充足 or open_questions化がされていること（黙って省略ゼロ）
- 曖昧語（きれいに/ちゃんと/なるべく/適宜）が出力に1つも残らないこと
- owner_check_pointsが全件2つ以下であること
