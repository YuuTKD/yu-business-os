# Claude Codeへの作業依頼 前置き安全指示文

> 本番系の作業をClaude Codeに依頼するとき、依頼文の冒頭にこのブロックを貼る。

```
【安全ルール（この作業で必ず守ること）】
1. 既存ファイルの削除・移動・上書きは禁止。必要と判断した場合は理由を提示して停止し、承認を待つこと
2. core/ configs/ Cloud Run実装は「新規追加」以外触らない
3. Secret / token / APIキー / .env / credentials / client_secret を、コード・コミット・出力のいずれにも含めない
4. git push・デプロイ・Scheduler変更・本番投稿は実行前に必ず停止し、pre-deploy-qaのチェック結果と一緒に確認を求めること
5. project_id / service_name / region を作業開始時に宣言し、途中で変えないこと
6. 作業完了時は「変更ファイル一覧・削除ゼロ確認・Secret混入ゼロ確認」を報告すること
```

## 依頼文の型
```
（上記の安全ルールブロック）

【作業内容】
- 目的：
- 対象リポジトリ／ブランチ：
- 変更してよい範囲：
- 完了条件：
- 停止して確認すべきポイント：デプロイ直前 / ファイル削除が必要になったとき
```

## 作業完了報告の受け取りチェック（人間側）
- [ ] 変更ファイル一覧が報告にある
- [ ] 削除ゼロ（またはゆうさん承認済みの削除のみ）
- [ ] Secret混入ゼロの確認文言がある
- [ ] デプロイが必要な場合、pre-deploy-qa未実施のまま実行されていない
