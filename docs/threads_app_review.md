## 1. 必要権限一覧（申請時に管理画面で確認してから記入）

| 権限名 | 種別 | 用途 |
|--------|------|------|
| `threads_basic` | Standard（審査不要） | 全エンドポイント共通・必須 |
| `threads_keyword_search` | **Advanced Access（審査必要）** | 沖縄グルメ関連の公開投稿をキーワード検索 |
| `threads_content_publish` | Standard（審査不要） | 承認済み返信の投稿公開 |
| `threads_manage_replies` | 要確認（本番は審査必要の可能性） | 第三者投稿への返信POST |

> **注意**: `threads_manage_mentions` が返信に必要な場合があります。申請前に公式ドキュメントで確認してください。

## 2. 各権限が必要な理由

### `threads_keyword_search` (Advanced Access)

**ユースケース**: 沖縄旅行中・旅行予定者が投稿する「沖縄おすすめご飯」「国際通りで食事」等のキーワードを含む公開投稿を収集し、当社が運営する飲食店（TACHINOMIYA・琉球火鍋）への来店支援情報を提供する。

**なぜ Advanced Access が必要か**: Standard Access では認証済みユーザー自身の投稿しか検索できず、旅行者が投稿した第三者の公開投稿を検索できないため。

**データの取り扱い**:
- 取得する情報: 投稿ID、投稿本文、投稿者名（公開情報のみ）、投稿日時、permalink
- 保存先: Google Spreadsheet（社内管理、非公開）
- 保持期間: 90日（来店確認後に削除）
- 第三者への提供: なし

### `threads_manage_replies`

**ユースケース**: スタッフが承認した投稿のみ、個別に生成した返信文を投稿する。

**なぜ必要か**: 第三者の投稿に返信するには `threads_manage_replies` または `threads_manage_mentions` が必要。

**安全設計**:
- 完全自動返信は禁止
- 人間スタッフの承認後のみ返信
- 1日最大10件・1時間最大3件の上限
- 同一投稿への返信は1回のみ
- 同一ユーザーへの返信は7日に1回のみ

## 3. ユーザー操作フロー（審査員向け）

```
[システム] 毎日定期実行（または手動実行）
    ↓
[API] GET /keyword_search?q=沖縄+おすすめ+ご飯&search_type=RECENT
    ↓
[判定] 関連性スコア計算（100点満点）
    70点以上 → 通知対象
    50〜69点 → 保留記録
    49点以下 → 除外
    ↓
[記録] Google Spreadsheet「THREADS_LEAD_MONITOR」シートに記録
    ↓
[通知] スタッフ用LINEへ承認依頼メッセージ送信
    （投稿本文・スコア・推奨店舗・返信案を含む）
    ↓
[スタッフ] スプレッドシートの「承認状況」列を更新
    「承認」→ 返信実行
    「見送り」→ 何もしない
    「修正」→ 返信案を編集してから承認
    ↓
[返信] POST /me/threads → threads_publish（承認済みのみ）
    ↓
[記録] 返信投稿ID・返信日時をシートに保存
```

## 4. 審査員向けテスト手順

### テスト環境へのアクセス

1. Meta Developer Console → アプリ → テスターとして追加（連絡先: yuya_tokuda@trees-catering.com）
2. テスターアカウントでログインし、アクセストークンを発行
3. テスト投稿を行い、以下のエンドポイントで動作確認

### テスト実行手順

**Step 1: キーワード検索確認**
```
GET https://graph.threads.net/v1.0/keyword_search
  ?q=沖縄+おすすめ+ご飯
  &search_type=RECENT
  &access_token={テスト用トークン}
```
期待結果: 公開投稿リストが返却される

**Step 2: 返信テスト（テスターアカウントの投稿に対して）**
1. テスターアカウントで「沖縄旅行中です。国際通りでおすすめのご飯はどこですか？」と投稿
2. システムが投稿を検知し、スタッフLINEに通知
3. スプレッドシートの承認状況を「承認」に変更
4. システムが返信を投稿
5. 投稿に返信が紐づいていることを確認

**Step 3: 重複防止確認**
- 同一投稿IDへ2回処理を実行 → 2回目はスキップされることを確認
- 同一ユーザーへ7日以内に再送信 → スキップされることを確認

## 5. 画面録画用シナリオ（審査提出用）

録画すべき操作シーケンス:

1. **キーワード検索** → テスト投稿が検索結果に含まれることを表示
2. **スプレッドシート確認** → 候補投稿・スコア・返信案が記録されていることを表示
3. **LINE通知** → スタッフへの承認依頼メッセージを表示（投稿本文・スコア・返信案含む）
4. **スタッフ承認操作** → スプレッドシートで「承認」を入力する様子
5. **返信投稿** → テスターアカウントの投稿に返信されることを確認
6. **重複防止** → 同一投稿を再処理してもスキップされることを表示
7. **DRY_RUNモード** → `THREADS_DRY_RUN=true` 設定時は実際に投稿されないことを表示
