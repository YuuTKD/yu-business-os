# 督促テンプレート Lv2：期限明示

**使用条件**: 支払期日から7〜14日経過  
**送信手段**: LINE（推奨）または メール  
**送信者**: 人間（AIは生成のみ・送信しない）  
**前提**: Lv1送信の記録がある場合が多い

---

## 送信テキスト

```
{{customer_name}} 様

お世話になっております。{{business_name}}の{{owner}}です。

先日ご連絡差し上げました、¥{{amount}} のお支払いについて、
本日時点でご入金の確認が取れておりません。

誠に恐れ入りますが、下記期限までにご対応いただけますでしょうか。

■ 支払期限：{{next_deadline}}（★人間が日付を設定★）
■ ご請求金額：¥{{amount}}
■ お振込先：[★人間が口座情報を追記★]

お支払い済みの場合は行き違いをお詫びいたします。
ご都合がある場合はお早めにご相談ください。

何卒よろしくお願いいたします。
```

---

## 差し込み変数

| 変数名 | 内容 | 必須 | 備考 |
|--|--|--|--|
| `{{customer_name}}` | 顧客名 | 必須 | |
| `{{business_name}}` | 事業名 | 必須 | |
| `{{owner}}` | 担当者名 | 必須 | |
| `{{amount}}` | 未収金額（円） | 必須 | カンマ区切り |
| `{{next_deadline}}` | 新しい支払期限 | 必須 | **人間が設定**（最低3営業日後推奨）|

---

## 人間確認チェックリスト（送信前に必ず確認）

- [ ] Lv1を既に送信しているか（DEBT_COLLECTION_LOG確認）
- [ ] `{{next_deadline}}` の日付は現実的か（短すぎない・長すぎない）
- [ ] 金額が正しいか
- [ ] 口座情報を追記したか
- [ ] 送信先が正しいか
- [ ] 顧客が支払い困難な事情を申告していないか（その場合は分割交渉を検討）

---

## 送信後の記録（人間が実施）

DEBT_COLLECTION_LOG の以下を更新：
- `M列（Lv2送信日）`：送信日を入力
- `O列（次回期限）`：`{{next_deadline}}` の値を入力
- `K列（状態）`：「Lv2送信」に変更
