■ 今回の問題
  fake_serp_realistic_mock 候補を本番LINE通知してしまった。
  全10件が架空アカウント・架空ドメイン・検索URLだった。

■ 再発防止の根本対策
  mock候補 → DRY_RUN専用（LINE本番通知禁止）
  本番通知 → URL_VERIFIED + スマホ実在確認済みのみ

==================================================
方法A: SerpAPI（有料・推奨）
==================================================
  費用: $50/月（100検索/月）
  仕組み: Google検索結果をAPI取得 → 実在URLのみ返却
  ただし: Instagram/Threadsは検索結果に出るが
          アカウント実在は依然スマホ確認が必要

  取得例:
    query: "沖縄 サロン SNS 続かない site:instagram.com"
    query: "沖縄 飲食店 Google投稿 放置 site:threads.net"

  実装:
    scripts/acquisition/run_product_match_acquisition_agent.js
    の ALLOWED_PROVIDERS に serpapi を設定
    SERPAPI_KEY を Cloud Run 環境変数に設定

==================================================
方法B: Google Custom Search Engine（無料枠あり）
==================================================
  費用: 無料（100クエリ/日）
  仕組み: Google CSE API でウェブ検索 → 実在URLを取得
  ただし: Instagram/Threadsの内部投稿は取得できない

  設定手順:
    1. Google Cloud Console で Custom Search API を有効化
    2. CSE (Programmable Search Engine) を作成
    3. GOOGLE_CSE_ID と GOOGLE_CSE_API_KEY を Cloud Run に設定

==================================================
方法C: ゆうさん手動入力（推奨・今すぐできる）
==================================================
  費用: 無料
  仕組み: ゆうさんがスマホで実在アカウントを見つけて
          手動でCSVに記入 → スクリプトがリプ文を生成

  入力ファイル: data/master/manual_candidates.csv
  列構成:
    candidate_name, url, platform, pain_signal,
    offer_type, verified_by, verified_at, note

  手順:
    1. Instagramで「沖縄 サロン AI 投稿続かない」などを検索
    2. 悩み投稿をしているアカウントを見つける
    3. manual_candidates.csv に記入
    4. python3 scripts/acquisition/gen_reply_sales.py を実行
    5. リプ文が生成される → LINEプレビューで確認

==================================================
方法D: Google Mapsで実在店舗を特定（無料）
==================================================
  仕組み: Google マップで実際に「那覇 整体院」等を検索
          → GBP投稿が古い・口コミ返信なしの店舗を特定
          → その店舗のInstagramアカウントを店舗ページから探す

  ただし: GBP経由の直接連絡は手動のみ
          SNSアカウントが見つかればリプ導線へ
